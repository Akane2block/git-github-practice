# Git/GitHub復習会 ハンズオンマニュアル

このリポジトリは、Git/GitHub復習会で使う練習用サイトです。
みんなで自己紹介カードを追加して、GitHubの基本操作を体験しましょう！

---

## 事前準備

以下が必要です：
- GitHubアカウント
- Cursor（エディタ）
- Gitのインストール

---

## ハンズオンの流れ

### Step 1: リポジトリをクローンする

1. Cursorを開く
2. `Cmd + Shift + P`（Mac）または `Ctrl + Shift + P`（Windows）でコマンドパレットを開く
3. 「Git: Clone」と入力して選択
4. 以下のURLを入力：
   ```
   https://github.com/Akane2block/git-github-practice.git
   ```
5. 保存先フォルダを選択
6. 「Open」でプロジェクトを開く

**ターミナルでやる場合：**
```bash
git clone https://github.com/Akane2block/git-github-practice.git
cd git-github-practice
```

---

### Step 2: 新しいブランチを作成する

**なぜブランチを作る？**
→ 自分の作業を他の人の作業と分けるため。失敗しても本体に影響しない。

1. Cursorの左下にある「main」をクリック
2. 「Create new branch...」を選択
3. ブランチ名を入力（例：`feature/tanaka-card`）
4. Enterで作成＆切り替え

**ターミナルでやる場合：**
```bash
git checkout -b feature/あなたの名前-card
```

---

### Step 3: 自己紹介カードを追加する

1. `index.html` を開く
2. 下の方にある「参加者のカードはこの下に追加」を探す
3. その下に、以下のテンプレートをコピペして編集：

```html
<div class="card">
    <div class="card-icon">🎉</div>
    <h2>あなたの名前</h2>
    <p class="role">ひとこと肩書き</p>
    <p class="message">自己紹介メッセージを書いてね！</p>
    <div class="likes">
        <span>好きなもの：</span>○○、△△、□□
    </div>
</div>
```

4. 自分の情報に書き換える
   - `card-icon`: 好きな絵文字に変更
   - `h2`: 自分の名前
   - `role`: 職種や一言
   - `message`: 自己紹介
   - `likes`: 好きなもの3つくらい

5. `Cmd + S`（Mac）または `Ctrl + S`（Windows）で保存

**確認方法：**
`index.html` をブラウザにドラッグ＆ドロップして表示を確認

---

### Step 4: 変更をコミットする

**コミットとは？**
→ 変更を記録すること。「セーブポイント」のようなもの。

1. Cursorの左側にある「ソース管理」アイコン（分岐マーク）をクリック
2. 変更されたファイル（`index.html`）が表示される
3. ファイル名の横の「+」をクリックしてステージング
4. 上部のテキストボックスにコミットメッセージを入力：
   ```
   自己紹介カードを追加（あなたの名前）
   ```
5. 「✓ コミット」ボタンをクリック

**ターミナルでやる場合：**
```bash
git add index.html
git commit -m "自己紹介カードを追加（あなたの名前）"
```

---

### Step 5: GitHubにプッシュする

**プッシュとは？**
→ ローカル（自分のPC）の変更をGitHub（クラウド）にアップロードすること。

1. Cursorの左下「↑」マーク（または「変更の同期」）をクリック
2. 初回は「Publish Branch」と表示されるのでクリック

**ターミナルでやる場合：**
```bash
git push -u origin feature/あなたの名前-card
```

---

### Step 6: プルリクエストを作成する

**プルリクエスト（PR）とは？**
→ 「私の変更をmainブランチに取り込んでください」というリクエスト。

1. GitHubのリポジトリページを開く
2. 「Compare & pull request」ボタンが表示されるのでクリック
3. タイトルを入力（例：「たなかの自己紹介カードを追加」）
4. 説明を書く（任意）
5. 「Create pull request」をクリック

---

### Step 7: レビュー＆マージ

**マージとは？**
→ ブランチの変更をmainブランチに統合すること。

1. 主催者（または他の参加者）がプルリクエストを確認
2. 問題なければ「Merge pull request」をクリック
3. 「Confirm merge」をクリック

---

### Step 8: 最新の状態をプルする

**プルとは？**
→ GitHub（クラウド）の最新状態をローカル（自分のPC）に反映すること。

1. Cursorでmainブランチに切り替える
   - 左下のブランチ名をクリック → 「main」を選択
2. 「↓」マーク（または「変更の同期」）をクリック

**ターミナルでやる場合：**
```bash
git checkout main
git pull
```

3. `index.html` を開いて、他の人のカードが追加されていることを確認！

## よく使うGit用語まとめ

| 用語 | 意味 |
|------|------|
| リポジトリ | プロジェクトの保管庫 |
| クローン | リポジトリをコピーしてダウンロード |
| ブランチ | 作業の分岐。本体に影響せず作業できる |
| コミット | 変更を記録する（セーブポイント） |
| プッシュ | ローカル→GitHubにアップロード |
| プル | GitHub→ローカルにダウンロード |
| プルリクエスト | 変更を取り込んでほしいというリクエスト |
| マージ | ブランチの変更を統合する |

---

## 困ったときは

- **「変更が反映されない」** → ブラウザをリロード（Cmd+R / Ctrl+R）
- **「プッシュできない」** → GitHubにログインしているか確認
- **「コンフリクトが発生した」** → 主催者に相談！
